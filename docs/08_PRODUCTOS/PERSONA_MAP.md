# 08.2 — Persona Map + Jobs-to-be-Done

> **Propósito:** 9 personas objetivo de DMX con Jobs-to-be-Done completos. Cada persona tiene aspirations, pain points actuales, outcomes deseados, DMX value prop, success metrics y acquisition channels.
> **Principio:** las personas definen productos (ver 08.3 PRODUCT_CATALOG). Un producto = conjunto de features (ver 08.1) que resuelve Job-to-be-Done específico de 1+ persona.
> **Última actualización:** 2026-04-20

---

## Framework Jobs-to-be-Done (JTBD)

Para cada persona documentamos:

1. **Profile** — quién es (edad, rol, industria, sofisticación tech, presupuesto)
2. **Aspiration** — qué sueña lograr (aspiracional)
3. **Current pain** — qué le duele hoy (problema concreto)
4. **Outcome buscado** — resultado medible que necesita
5. **DMX solution** — qué features resuelven el JTBD (IDs de 08.1)
6. **Success metric** — cómo medimos que funcionó
7. **Time-to-value** — cuánto tarda usuario en obtener valor
8. **Acquisition channel** — dónde encontrarlo + mensaje
9. **Pricing fit** — qué tier puede pagar

## 🏗️ Persona 1 — Developer / Desarrollador

### Profile
- Propietario/director empresa desarrolladora inmobiliaria MX
- Tamaño empresa: pequeña (5-50 unidades/año) a grande (500+)
- Presupuesto IT: $50K-$500K MXN/año en herramientas
- Sofisticación tech: media-alta
- Edad típica: 35-55

### Aspiration
"Construyo proyectos rentables rápido, maximizo margen y ROI, minimizo riesgo de absorción lenta o precios mal calibrados."

### Current pain
- "No sé si la zona X realmente tiene demanda para el producto que pienso construir"
- "Tengo terreno pero no sé qué debo construir ahí (mix unidades, amenidades, precios)"
- "Mi competencia lanza productos similares y no sé cómo diferenciarme"
- "Mis precios los pongo por intuición, no por data"
- "Dependo de consultorías caras que tardan meses (Brandata, Softec)"

### Outcome buscado
- Identificar las 3-5 mejores zonas CDMX/LATAM para construir según mi portfolio + thesis
- Definir mix óptimo unidades/amenidades para cada proyecto
- Calibrar pricing dinámico por unidad (no precio lista estático)
- Monitor competencia en tiempo real
- Reducir absorción promedio 20-30%

### DMX solution
**Features core:**
- S-N11 DMX Momentum Index (zonas trending)
- S-B01 Demand Heatmap + S-B07 Competitive Intel + S-B08 Absorption Forecast
- S-B03 Pricing Autopilot + S-B09 Cash Flow + S-B13 Amenity ROI (N2)
- S-B06 Project Genesis + S-H13 Site Selection AI (N3)
- S-B15 Launch Timing + S-B14 Buyer Persona Proyecto (N2)
- S-E01 Full Project Score + S-E02 Portfolio Optimizer (N4)
- S-I01 DMX Estimate AVM

**Laterales clave:**
- L-60 LandAnalyzer Wizard (vocación predio automatizada)
- L-50 Crisis Alerting Zones
- L-32 DMX Zone Certified (mejorar scores = visibilidad)
- L-43 Social Radar Dashboard (monitor competencia)
- L-70 Space Optimization layout

### Success metric
- Time-to-market reducido 30-50% vs consultorías tradicionales
- ROI proyectos +15-25% vs baseline (pricing optimizado)
- Costo data intelligence reducido 60-80% vs Brandata/DatAlpine consultoría
- Absorción promedio <180 días (vs industry avg 300+)

### Time-to-value
- Inmediato (día 1): onboarding zona target + vista scores actuales
- 1 semana: LandAnalyzer genera viability report terreno X
- 1 mes: Pricing Autopilot ajusta precios unidades activas
- 3 meses: Portfolio Optimizer sugiere reallocation capital

### Acquisition channel
- LinkedIn Ads targeted "Directores desarrollo inmobiliario MX"
- Ferias inmobiliarias (Expo Real Estate CDMX)
- Referencias PM clients (consulting → SaaS conversion)
- Content marketing: blog casos uso + whitepapers con data DMX

### Pricing fit
- **Starter**: Pro tier $599-$899 USD/mes (5-50 unidades/año)
- **Pro**: Developer tier $1,199-$1,999 USD/mes (50-200 unidades)
- **Enterprise**: Custom $50K-$200K MXN/año + revenue share opcional
