# 07.0 — Índice del Catálogo de Game-Changers

> **110 game-changers catalogados** — organizados en 7 tiers cross-industry, 3 perfiles de rol day-to-day (asesor, comprador, desarrollador) y cross-industry misc. Cada game-changer tiene ID estable `GC-NN` (GC-1 a GC-110) y se referencia por ese ID en todas las fases, ADRs y documentos downstream.

---

## Contexto

La biblia v2 moonshot parte de una convicción: **DesarrollosMX no compite contra Inmuebles24 ni Vivanuncios**. Compite contra el futuro de cómo se compra, vende, desarrolla y financia el real estate en LATAM. Y ese futuro exige robar ADN — sin imitarlo servilmente — de las 25+ empresas que ya ganaron en sus categorías: Stripe (API-as-product), Anthropic (Constitutional AI), Palantir (forward-deployed + ontology), Tempus (precision data), Airbnb (marketplace + superhost), Nvidia (compute + framework), Robinhood (fractional), Nu (embedded banking), Coinbase (compliant tokenization), Shopify (app ecosystem), Clay / Lusha / Apollo / Manus.im (enrichment + sequencing + research), Duolingo / Strava / Spotify / MasterClass (gamification + retention), TikTok / Discord / Roblox / Figma (network + community + multiplayer), Notion / Slack / Airtable / Loom / Zapier (productivity workflows), Substack / Bloomberg / WSJ (distribution + media), y Zillow + Pulppo para las mecánicas reales del sector.

El resultado del análisis cross-industry es este catálogo de 110 game-changers. **No todos se construyen — no todos se construyen ahora**. Este documento es el catálogo maestro; la priorización H1 / H2 / H3 vive en `07.3_PRIORIZACION_H1_H2_H3.md`; el análisis de moat vive en `07.4_MOAT_STRATEGY.md`; los patrones fundacionales cross-industry viven en `07.2_CROSS_INDUSTRY_PATTERNS.md`.

## Cómo leer el catálogo

Cada GC tiene la misma estructura:

```markdown
### GC-NN — [Nombre corto]
**Rol / Tier:** [Asesor | Comprador | Desarrollador | Tier X]
**Inspiración:** [empresa / patrón fuente]
**Pain point:** [1 línea — el problema real en el mercado LATAM hoy]
**Descripción:** [2-3 líneas — qué construimos y cómo resuelve el pain]
**Fase propuesta:** FASE XX (H1 | H2 | H3)
**Dependencias técnicas:** [APIs, ADRs, fases previas]
**Moat factor:** [bajo | medio | alto + 1 línea de justificación]
```

Los GCs de `07.2` incluyen además un campo adicional:

```markdown
**Inspiration company analysis:** [1 párrafo — por qué esa empresa ganó su categoría y qué patrón robar]
```

## Criterios de inclusión en el catálogo

Un GC entra al catálogo si cumple **todos** estos criterios:

1. **Evidencia empírica** — existe una empresa (o varias) que ya lo hizo funcionar en otra industria o en real estate adyacente.
2. **Aplicabilidad a DesarrollosMX** — se puede mapear al contexto LATAM + stack + alcance del producto.
3. **Defensibilidad contextual** — no es un feature trivial que cualquier competidor puede copiar en un sprint; o si lo es, su valor viene de la composición con otros GCs.
4. **Unidad atómica** — describe una capacidad, no un épico. Si es un épico, se desglosa.

Criterios de exclusión:

- No está en el scope del producto (ej: construir un LLM propio — se consume, no se entrena).
- No es replicable en LATAM por razones regulatorias o estructurales (ej: MLS-only features de USA).
- Es puro marketing, no producto.

## Relación con el plan maestro

El plan maestro `docs/02_PLAN_MAESTRO/FASE_XX.md` es la **autoridad final** sobre qué se construye y cuándo. Este catálogo alimenta el plan, no lo reemplaza. ## Glosario mínimo

- **GC** — Game-Changer. Capacidad única del producto (no épico).
- **H1 / H2 / H3** — Horizonte 1 / 2 / 3 según ADR-011. Aprox H1 = 0-12 meses, H2 = 12-30 meses, H3 = 30-60 meses.
- **RICE** — Reach × Impact × Confidence ÷ Effort. Scoring de priorización.
- **Moat** — Ventaja competitiva defensible que no se erosiona con tiempo/dinero del rival.
- **IE** — Índice de Empresabilidad (score propietario DMX que combina 6 cascadas de señales).
- **Cascada** — Pipeline de scoring (ver ADR-010).
