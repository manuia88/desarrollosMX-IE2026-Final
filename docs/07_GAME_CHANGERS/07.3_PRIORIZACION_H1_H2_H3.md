# 07.3 — Priorización H1 / H2 / H3

> Matriz de priorización de los **110 game-changers** en los 3 horizontes definidos por [ADR-011](../01_DECISIONES_ARQUITECTONICAS/ADR-011_MOONSHOT_3_HORIZONTES.md). **H1 = 24, H2 = 40, H3 = 46**. Total = 110. Check.

---

## 1. Intro y criterios de priorización

El catálogo de 110 GCs es el universo de capacidades; el plan operativo requiere priorizarlas. Usamos una variante de **RICE adaptada a DesarrollosMX**:

| Criterio | Descripción | Peso |
|---|---|---|
| **Reach** | # usuarios afectados (asesores / compradores / desarrolladores / B2B) | 20% |
| **Impact** | Impacto en moat + revenue + retention (escala 1-5) | 30% |
| **Confidence** | Evidencia en empresas fuente (escala 1-5, 5 = Stripe-level proof) | 15% |
| **Effort** | Ingeniería + ops (en semanas-persona) — invertido | 15% |
| **Strategic alignment** | Encaje con ADR-011 y los 3 horizontes | 15% |
| **Dependencies** | Número y madurez de prereqs (menos = mejor) | 5% |

**Fórmula:**

```
Score = (Reach × 0.20) + (Impact × 0.30) + (Confidence × 0.15) + (1/Effort × 0.15) + (Strategic × 0.15) + (1/Dependencies × 0.05)
```

**Cut-offs:**
- H1: Score ≥ 4.0 + alineado con `Producto base operacional` (ADR-011).
- H2: Score 3.0-4.0 + alineado con `Plataforma + ecosystem`.
- H3: Score < 3.0 o dependencies > 3 fases fuera + alineado con `Moonshot diferenciador`.

> Nota: los scores exactos están en un spreadsheet interno — este documento sólo consolida la asignación final, no la matemática completa.

### Definición de horizontes (recap ADR-011)

| Horizonte | Enfoque | Duración aprox |
|---|---|---|
| **H1** | Producto base operacional. Validar product-market fit en 1-2 países. 24 GCs. | 0-12 meses |
| **H2** | Plataforma + ecosistema. Expansión multi-country, apps, terminal, community. 40 GCs. | 12-30 meses |
| **H3** | Moonshot diferenciador. Moats durables (digital twin, agentic marketplace, banking, tokenization). 46 GCs. | 30-60 meses |
