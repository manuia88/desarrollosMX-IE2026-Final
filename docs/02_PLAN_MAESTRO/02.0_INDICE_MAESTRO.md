## Resumen ejecutivo

Este plan maestro describe la construcción de **DesarrollosMX** (DMX) — una plataforma de Spatial Decision Intelligence para el sector inmobiliario LATAM — desde cero, en **39 fases** atómicas y verificables, agrupadas en tres horizontes (H1 launch + MVP platform, H2 platform play + agentic + digital twin + data ecosystem + creator + terminal, H3 fractional investing + embedded banking + international expansion). El plan es **AI-native** (Copilot + Command Palette + voice como UI primaria), **multi-country desde día 1** (MX/CO/AR/BR/CL + Latinx US en H3), y está diseñado para ejecutarse con Claude Code operando agentes paralelos sobre un stack Next.js 16 + Supabase + tRPC 11. El público primario es el equipo de desarrollo (founder + agentes AI + devs humanos futuros); el secundario, auditores técnicos, inversionistas y cualquier stakeholder que necesite entender qué se construye y por qué. Para navegar: empieza por [`00_FOUNDATION/`](../00_FOUNDATION/), consulta los [ADRs](../01_DECISIONES_ARQUITECTONICAS/) cuando toques áreas críticas, y sigue las fases en el orden marcado por el DAG de dependencias más abajo.

> *"El marketplace es el canal. El IE es el producto. Los datos temporales acumulados son el moat."* — Manu Acosta (founder)

## Cómo ejecutar el plan

1. **Leer [`00_FOUNDATION/`](../00_FOUNDATION/) en orden** — visión, stack, glosario, comandos. Sin esto nada más tiene sentido.

5. **Si una decisión no está documentada, crear ADR antes de construir.** Plantilla Nygard — ver ADR existentes como referencia.
6. **Runbook para new devs:** [`05.4_ONBOARDING_DEV.md`](../05_OPERACIONAL/05.4_ONBOARDING_DEV.md).
