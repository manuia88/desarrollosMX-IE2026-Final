## 9. API externa v1 (B2B)

Middleware común: `validateApiKey → checkRateLimit → logRequest → formatResponse(plan)`.
Autorización via header `x-dmx-api-key` o Bearer token.

### 9.1 `GET /api/v1/scores/livability?lat=&lon=[&radius_m=500]`
| Propósito | DMX Livability API (Producto 1). Score 0-100 + breakdown + confidence. |
| Auth | API key plan free/starter/pro/enterprise |
| Rate limit | 100/día (free), 10K/día (starter), 100K/día (pro), unlimited (enterprise) |
| Response schema | `{ lat, lon, score: {livability, breakdown:{walkability, safety, ecosystem, water, schools, health, senior, transit}}, confidence, currency_context, disclaimer }` |
| Fase | FASE 23 |

### 9.2 `GET /api/v1/scores/momentum?zone_id=...&period=monthly`
DMX Momentum Index (Producto 2). Response: `{ zone_id, period, value, rank, trend_6m, components }`.

### 9.3 `GET /api/v1/scores/risk?lat=&lon=`
DMX Risk Score (Producto 3). `{ risk_score, components: {seismic, flood, water_infra, crime_trend, land_use}, recommendations }`.

### 9.4 `GET /api/v1/scores/project?project_id=`
Score integral proyecto.

### 9.5 `GET /api/v1/scores/[scoreType]?entity=&entity_id=`
Genérico por score_code.

### 9.6 `POST /api/v1/estimate`
DMX Estimate I01 (AVM). Body `{ address?, lat, lon, m2, recamaras, banos, amenities?, age_years? }`. Response `{ estimate, range_low, range_high, confidence, comparables, adjustments, market_context }`. Free 5/mes, Pro $2,999, API $15K-$50K.

### 9.7 `POST /api/v1/site-selection`
DMX Site Selection (Producto 4). Body `{ criteria: {target_persona, tipo_propiedad, max_budget_m2, required_amenities, avoid_risks, min_transit_score} }`. Response: top 20 zonas scored.

### 9.8 `GET /api/v1/reports/market?zone_id=&period=`
DMX Market Intelligence Reports (Producto 5). Response PDF URL signed. $5K–$20K por reporte.

### 9.9 `GET /api/v1/rankings?index_code=&country_code=`
Top colonias por índice DMX.

### 9.10 `GET /api/v1/neighborhood-report?zone_id=`
Reporte zona (JSON). Compone todos los scores zonales.

### 9.11 `POST /api/v1/compass` (H3)
DMX COMPASS — predicción cierre operación.

### 9.12 `POST /api/v1/knowledge-graph/query` (H3)
DMX Knowledge Graph query.

## Schemas TS compartidos

Ubicación: `shared/schemas/api-v1.ts`. Ejemplos:
```ts
export const livabilityRequest = z.object({
  lat: z.number().min(-90).max(90),
  lon: z.number().min(-180).max(180),
  radius_m: z.number().int().min(100).max(2000).default(500),
});

export const livabilityResponse = z.object({
  lat: z.number(), lon: z.number(),
  score: z.object({
    livability: z.number().min(0).max(100),
    breakdown: z.record(z.number().min(0).max(100)),
  }),
  confidence: z.enum(['high','medium','low','insufficient_data']),
  version: z.string(),
  generated_at: z.string().datetime(),
});
```

## Notas / disclaimer

- Cada route en `/api/v1/*` tiene SLO <200ms p95; usar Vercel KV/Upstash cache para scores hot paths.
- Runtime `edge` donde aplique (`/api/ai/*`, `/api/v1/scores/*` read-only).
- Toda request de API externa se loggea en `api_request_logs` (`endpoint, status, response_ms`).
- Los webhooks inbound deben responder 200 en <3s; offloadear a Trigger.dev task.
- CSP estricto (ADR-009): solo dominios explícitos. Webhooks se verifican con HMAC antes de encolar.
