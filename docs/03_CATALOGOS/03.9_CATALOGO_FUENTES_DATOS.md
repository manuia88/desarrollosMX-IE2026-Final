**Mapeo SCIAN propietario (IE1-§3.18):**
- **3 tiers**: premium (33 códigos: restaurantes premium, consultorios especializados, cafeterías boutique, galerías, coworking, etc.), standard (~80: supermercados, farmacias, bancos, escuelas privadas, gimnasios), basic (~30: tianguis, tlapalerías, talleres, vulcanizadoras).
- **12 macro_categories**: ALIMENTACION, SALUD, EDUCACION, SERVICIOS_PROF, COMERCIO_RETAIL, GASTRONOMIA, ENTRETENIMIENTO, BELLEZA_PERSONAL, TECNOLOGIA, TRANSPORTE, MANUFACTURA, GOBIERNO.
- **Staff estimate mid-point**: `0_5→3, 6_10→8, 11_30→20, 31_50→40, 51_100→75, 101_250→175, 251_plus→350`.
- **Ratio Premium/Basic**: >2 consolidada, 1-2 transición, <1 popular.
- **Shannon-Wiener H**: `H = -Σ(p_i × ln(p_i))`. >2 diverso, 1.5-2 moderado, <1.5 poco diverso.
- **Gentrification Velocity**: `Δ(ratio_premium_basic) / Δ(meses)` requiere ≥2 snapshots separados ≥3 meses.

## Notas / disclaimer

- Toda fuente nueva debe registrarse con: `source_code` único, `license` (CC-BY/open/priv), `refresh_frequency`, `last_success_at`, `owner` (en tabla interna `data_sources_registry` H2).
- Los ingestores se implementan como Trigger.dev tasks con retry + dead letter (ver catálogo 03.7).
- El mapeo SCIAN propietario vive en `features/ingesta/scian-mapping.ts`. Se considera **IP propietaria crítica** (no open-source).
- **Hard rule**: cualquier ingestor nuevo debe tener tests unitarios de parsing + snapshot de 1 semana mock data antes de ship.
- Multi-country prioritization: MX H1 (launch), CO H2 (M+6), AR H2 (M+9), BR H3 (M+12), CL H3 (M+12).
