## Cómo usar este documento

Este documento es **vivo**: cada PR que añade o modifica UI interactiva debe **añadir o actualizar filas** en la matriz correspondiente a la fase.

**Columnas estándar:**

| Columna | Qué describe |
|---------|--------------|
| **UI Component** | Path + component name + identificador (ej. `features/contactos/components/ContactCard.tsx:QuickCallButton`). |
| **Route** | App Router path donde vive el componente. |
| **Handler** | Nombre de la función / prop que se dispara (ej. `handleCall`, `onSubmit`). |
| **tRPC Procedure** | `router.procedure` invocado (ej. `contactos.logCall`). |
| **DB Op** | Tipo de operación + tabla afectada (ej. `INSERT interaction_feedback`). |
| **Cascades/Triggers** | Triggers BD que disparan (ej. `trigger_update_contacto_last_interaction`). |
| **Audit Log** | Event type en `audit_log` (ej. `contactos.call_logged`). |
| **Notifs** | Tipo de notificación (clave de 03.12 — ej. `notif-call-logged`). |
| **Permissions** | Rol + feature gating (ej. `asesor` + `feature:call_tracking`). |
| **Tests** | Path del test Playwright/unit (ej. `tests/e2e/contactos/quick-call.spec.ts`). |
| **Status** | `active` / `stub` / `deprecated`. Si `stub`, comentario `// STUB — FASE XX` obligatorio. |

## Convenciones

### UI Component
Formato: `<feature-path>:<ComponentName>[:<identifier>]`.

Ejemplos:
- `features/contactos/components/ContactCard.tsx:QuickCallButton`
- `features/dashboard/command-center/ActionButtons.tsx:CreateTaskBtn`
- `app/auth/login/page.tsx:LoginForm`

### Route
App Router path literal. Para rutas dinámicas usar brackets del framework: `/crm/contactos/[id]`.

### Handler
Nombre exacto de la función disparada. Debe ser no-op-free (ver ADR-018 Regla 1).

### tRPC Procedure
Formato `router.procedure`. Si el handler llama a múltiples procedures, listar todos separados con `+`: `contactos.logCall + notifs.emit`.

### DB Op
Tipo + tabla. Múltiples ops en misma fila separadas con `+`:
- `INSERT contactos + UPDATE asesor_gamification`.

### Cascades/Triggers
Nombres de triggers BD registrados en 03.3 (CATALOGO_BD_TRIGGERS). Si no hay cascadas, poner `—`.

### Audit Log
Event type. Todos los mutations CRUD significativos (ADR-018 Regla 7) deben emitir event. Si no aplica: `—`.

### Notifs
Clave del catálogo 03.12 (NOTIFS_Y_WEBHOOKS). Si no aplica: `—`.

### Permissions
`<rol>` o lista + feature gate. Ejemplos:
- `asesor`
- `asesor + feature:bulk_import_contacts`
- `admin + MFA required`
- `public` (sin auth, solo para `publicProcedure`).

### Tests
Path del archivo Playwright + test name si aplica. Formato: `tests/e2e/<fase>/<scenario>.spec.ts:<test_name>`.

### Status
- **`active`**: implementado, testeado, en producción.
- **`stub`**: marcado con las 4 señales (ADR-018): comentario código + badge UI + §5.B + 501.
- **`deprecated`**: fue removido. Mantener fila por historial, fecha de deprecation.

## Auditoría y CI

### Script `scripts/audit-dead-ui.mjs`

Herramienta AST parser (Node 24 + `@typescript-eslint/parser` + `@typescript-eslint/visitor-keys` + `fast-glob`) que escanea todos los `.tsx` bajo `features/**/*.tsx` y `app/**/*.tsx` detectando violations de ADR-018.

**Patterns detectados (exit code 1 si cualquiera detectado):**

1. **`<Button>` sin `onClick`** (excepto `disabled` o `type="submit"` dentro de `<form onSubmit>`).
2. **`<form>` sin `onSubmit`**.
3. **Handler vacío** `onClick={() => {}}` o `onClick={() => undefined}` o `onClick={noop}`.
4. **Handler placeholder** `onClick={() => alert(...)}` o `onClick={() => console.log(...)}`.
5. **`href="#"` o `href=""`** en `<Link>` o `<a>`.
6. **`useEffect` sin deps** (hook que hará fetch sin dependencies — warning).
7. **`useQuery` que referencia procedure que no existe en `appRouter`** (cross-ref con types).
8. **tRPC procedures que `throw new Error('TODO')`** sin comentario `// STUB —`.
9. **Procedures que `return hardcoded_obj`** sin comentario `// STUB —` y que no están marcadas como mock test.
10. **Componentes importando tRPC hook pero no usándolo** (dead import — se remueve).

**Output:**
```json
[
  {
    "file": "features/agents/components/AgentList.tsx",
    "line": 42,
    "column": 16,
    "pattern": "button_no_onclick",
    "message": "Button without onClick handler. Add handler, add disabled, or mark as STUB.",
    "severity": "error"
  }
]
```

**Invocación:**
```bash
npm run audit:dead-ui                    # local con nice output
npm run audit:dead-ui -- --ci --json    # para CI como JSON
```

**CI workflow** (`.github/workflows/e2e-audit.yml`):
```yaml
name: E2E Audit
on: [pull_request]
jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '24' }
      - run: npm ci
      - run: npm run audit:dead-ui -- --ci --json > audit-report.json
      - if: failure()
        uses: actions/upload-artifact@v4
        with: { name: audit-report, path: audit-report.json }
```

Configurado como **required status check** en branch protection rules `main`.

## Política de STUBs permitidos

Per ADR-018, un STUB está permitido **solo si cumple las 4 señales simultáneas**:

### Señal 1 — Comentario en código
```ts
// STUB — activar FASE 31 con [dep: LangGraph + Agent templates]
export const run = authenticatedProcedure
  .input(agentRunInput)
  .mutation(() => {
    throw new TRPCError({ code: 'NOT_IMPLEMENTED', message: 'agents.run disponible en FASE 31. Ver ADR-014.' });
  });
```

### Señal 2 — UI badge visible al user
```tsx
<Button disabled aria-label="Próximamente FASE 31">
  Ejecutar agente
  <Badge variant="soon">próximamente</Badge>
</Button>
```
Badges permitidos: `[beta]` | `[próximamente]` | `[alpha]` | `[preview]`.

### Señal 3 — Documentado en §5.B de la fase actual
En el `FASE_NN.md` correspondiente, sección `## 5.B — Inferencias y stubs permitidos`:
```markdown
- `agents.*` — stubs marcados 501 NOT_IMPLEMENTED. Activación FASE 31. Depende: LangGraph + Agent templates.
```

### Señal 4 — Endpoint devuelve 501
No 200 fake con payload hardcoded. Siempre:
```ts
throw new TRPCError({ code: 'NOT_IMPLEMENTED', message: '...' });
```

**Si falta cualquiera de las 4 → el stub es ilegal → audit-dead-ui falla → PR no mergea.**
