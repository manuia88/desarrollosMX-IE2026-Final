## Middleware compartido

Toda request pasa por:
```
validateApiKey → checkRateLimit → logRequest → formatResponse(plan)
```

### `validateApiKey`
- Header `x-dmx-api-key` o `Authorization: Bearer <token>`.
- Busca por hash SHA-256 en `api_keys`.
- Verifica `active=true`, `plan in ('free','starter','pro','enterprise')`.
- Carga plan into context.

### `checkRateLimit`
- Sliding window 60s en `api_rate_limits`.
- Limits por plan: free=1/s, starter=10/s, pro=100/s, enterprise=1000/s.
- Headers response: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.

### `logRequest`
- Inserta en `api_request_logs` asíncrono (edge case: no bloquear response).
- Fields: endpoint, status_code, response_ms, ip, api_key_id.
- Incrementa `api_keys.requests_today` y `requests_month`.

### `formatResponse(plan)`
- Free: solo `{score, confidence, version}` sin breakdown.
- Starter: +breakdown + components.
- Pro: +historical + trends + bulk.
- Enterprise: raw + metadata + custom fields.

## Producto 1 — DMX Livability API

### Overview
Score compuesto de habitabilidad 0-100 + breakdown por dimensiones. Basado en índice DMX-LIV.

### Endpoint
`GET /api/v1/scores/livability?lat={lat}&lon={lon}&radius_m={100-2000}`

### Request schema
```ts
const livabilityRequest = z.object({
  lat: z.number().min(-90).max(90),
  lon: z.number().min(-180).max(180),
  radius_m: z.number().int().min(100).max(2000).default(500),
  country_code: z.string().length(2).optional().default('MX'),
  persona: z.enum(['quiet','nightlife','family','fitness','remote_worker','investor']).optional(),
});
```

### Response schema
```ts
{
  lat: 19.4326, lon: -99.1332, radius_m: 500,
  score: {
    livability: 82.3,
    breakdown: {
      walkability: 85.2,  // N08
      safety: 78.1,       // F01
      ecosystem: 88.5,    // N01
      water: 72.4,        // N07
      schools: 81.0,      // H01
      health: 76.8,       // H02
      senior_friendliness: 68.2,  // N10
      transit: 92.1,      // F02
      crime_trend: 75.5,  // N04
    },
  },
  confidence: 'high',   // 'high' | 'medium' | 'low' | 'insufficient_data'
  currency_context: { currency: 'MXN' },
  disclaimer: 'Metodología pública en desarrollosmx.com/metodologia',
  version: '1.0.0',
  generated_at: '2026-04-17T10:15:00Z',
}
```

### Target clientes
- **Portales**: Inmuebles24, Vivanuncios (widget en ficha).
- **Fintechs**: Creditas, Coru, Yotepresto (scoring crediticio adicional).
- **Apps movilidad**: Uber, Cabify (heatmap destino).
- **Gobierno**: SEDUVI, INVI (planeación).
- **Aseguradoras**: GNP, MetLife (pricing pólizas).

### Pricing
| Plan | Precio/mes MXN | Queries/día | Incluye |
|---|---|---|---|
| Free | $0 | 100 | score + confidence |
| Starter | $5,000 | 10K | + breakdown + components |
| Pro | $25,000 | 100K | + historical + trends + bulk 100/call |
| Enterprise | Custom (desde $50K) | Ilimitado | SLA 99.9%, dedicated, custom endpoints |

### SLA H1
- p95 < 200ms.
- Uptime 99.5% (starter), 99.9% (pro+enterprise).

### Overview
AVM (Automated Valuation Model) mexicano. Inspirado en Zillow Zestimate + HouseCanary Value by Conditions.

### Endpoint
`POST /api/v1/estimate`

### Request
```ts
{
  address?: string,
  lat: number, lon: number,
  m2_totales: number,
  m2_construccion?: number,
  recamaras: number,
  banos: number,
  estacionamientos?: number,
  amenities?: string[],
  age_years?: number,
  tipo_propiedad: 'vertical'|'horizontal',
  condition?: 'nueva'|'remodelada'|'seminueva'|'usada'|'regular',
  country_code: 'MX',
}
```

### Response
```ts
{
  estimate: 4_850_000,
  range_low: 4_620_000,
  range_high: 5_100_000,
  currency: 'MXN',
  confidence: 0.78,              // 0-1
  comparables: [
    {address, m2, precio, distance_m, weight},
    // ...
  ],
  adjustments: {
    amenity_premium: 0.08,
    age_discount: -0.04,
    location_score: 0.15,
  },
  market_context: {
    zone_momentum: 0.78,
    zone_livability: 82.3,
    absorption_6m: 0.20,
  },
  version: '1.0.0-h1-linear',   // h2: gradient-boosting, h3: deep-learning
  generated_at: '...',
```

- `geo_data_points` (DENUE, GTFS, Atlas)
- `operaciones` (calibración transacciones — Tier 4)
- Características propiedad
- Condiciones específicas

### 3 fases de desarrollo
- **H1**: Regresión lineal + comparables ponderados por distancia y similitud.
- **H2**: Gradient boosting (XGBoost) con 47 features.
- **H3**: Deep learning calibrado con transacciones reales (requiere ≥100 ventas).

## Producto 9 — DMX Agent Marketplace (FASE 31)

### Overview
Marketplace de agentes 24/7 que ejecutan tareas autónomas para asesores/devs/compradores. Patrón LangGraph + durable execution (Vercel WDK). Creadores (power asesores, desarrolladores) publican agentes; consumers los instalan. Revenue share 70/30 creador/platform.

### Ejemplos de agentes
- `night-hunter` — cada noche 2am busca nuevas oportunidades matching criterios X → envía top 3 con DMX Score al Slack del asesor.
- `competitor-radar` — monitorea 5 proyectos competidores; alerta cuando hay cambio precio > 5%, nueva unidad, o campaña marketing.
- `deal-reviewer` — revisa nueva operación propuesta, valida compliance, comisiones, documentos; devuelve pass/fail con razones.
- `comprador-matcher` — cada vez que hay un contacto nuevo en CRM, matchea con inventory en radio 5km + envía top 5.
- `buyer-lifestyle-dna` — para comprador, analiza historial clicks + favoritos + visitas → refina DNA lifestyle weekly.

### Surface
- App `/agents` en portal asesor/dev/comprador (CRUD + observabilidad + marketplace público).
- API `/api/v1/agents/*` (create, run, schedule, observability — ver 03.5 §22).
- Marketplace público `https://desarrollosmx.com/agents` (discover, preview, install).

### Target clientes
- Asesores Pro/Enterprise — agentes personales.
- Desarrolladoras — agentes de inventory management.
- Master Brokers — agentes de supervisión team.
- Terceros creadores (power users) — publican agentes para cobrar.

### Pricing

|---|---|---|

| Pro | Incluido en plan Pro asesor | 5 agents, 500 runs/mes, agentes marketplace gratis |
| Enterprise | Incluido en plan Enterprise + custom | Unlimited, custom agents, private marketplace |

### Revenue model

## Producto 13 — DMX Fractional SPV (FASE 36)

### Overview

- App `/investing` portal comprador / inversor.

## Producto 14 — DMX Embedded Banking (FASE 37)

### Overview
Banking-as-a-Service (BaaS) embebido en DMX platform. Patrón Nu México / Revolut. Asesores + brokers + clientes post-operación pueden abrir cuenta DMX con tarjeta débito, transferencias SPEI, savings con yield, crédito preaprobado basado en DMX Score. Revenue via interchange + savings spread + crédito.

### Features clave
- Cuenta DMX con CLABE + tarjeta débito virtual/física.
- Transferencias SPEI gratis ilimitadas.
- Savings account con yield competitivo (target 9-11% anual H3).
- Crédito hipotecario preaprobado con tasa mejor mercado (patrocinado por bancos partners del Data Ecosystem).
- Insurance products embebidos (seguros GNP/MetLife con pricing dinámico DMX Risk Score).
- Remittances CO/AR/BR/CL via partners Wise/Belvo.

### Surface
- App `/banking` portal integrado (asesor + comprador + post-operación clients).
- Mobile apps iOS/Android.
- API `/api/v1/banking/*` (TBD FASE 37).

### Target clientes
- Asesores power users (captan comisiones → guardar en savings DMX).
- Brokers (manage team payroll + expenses).
- Clientes post-operación (recién compraron casa, abren cuenta para gastos relacionados).
- Inversores fractional SPV (cash management integrado).

### Pricing / Revenue model
- **No subscription fee** (cuenta gratis).
- **Interchange**: ~1.5% en cada transacción debit card (industry standard).
- **Savings spread**: DMX paga 9% al user, coloca a 12%, captura 3% spread.
- **Credit origination**: 1-2% fee sobre créditos originados (split con banco partner).
- **Foreign exchange**: 0.5-1% spread en remittances.
- **Overdraft/late fees**: aplicable según condiciones.

### Revenue target H3
$500K - $20M MXN/mes (depende de cuentas activas + AUM + crédito volume. Proyección 50K cuentas activas mes 60).

### Compliance
- Alianza con banco patrocinador (Broker-dealer + SOFOM regulada).
- CNBV compliance.
- Ley Fintech + LRAF.
- KYC/AML stack (Mati + Sumsub + TRM Labs sanctions).

### GC / ADR refs
