## Middleware compartido

Toda request pasa por:
```
validateApiKey → checkRateLimit → logRequest → formatResponse(plan)
```

### `validateApiKey`
- Header `x-dmx-api-key` o `Authorization: Bearer <token>`.
- Busca por hash SHA-256 en `api_keys`.
- Verifica `active=true`, `plan in ('free','starter','pro','enterprise')`.
- Carga plan into context.

### `checkRateLimit`
- Sliding window 60s en `api_rate_limits`.
- Limits por plan: free=1/s, starter=10/s, pro=100/s, enterprise=1000/s.
- Headers response: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.

### `logRequest`
- Inserta en `api_request_logs` asíncrono (edge case: no bloquear response).
- Fields: endpoint, status_code, response_ms, ip, api_key_id.
- Incrementa `api_keys.requests_today` y `requests_month`.

### `formatResponse(plan)`
- Free: solo `{score, confidence, version}` sin breakdown.
- Starter: +breakdown + components.
- Pro: +historical + trends + bulk.
- Enterprise: raw + metadata + custom fields.

## Producto 1 — DMX Livability API

### Overview
Score compuesto de habitabilidad 0-100 + breakdown por dimensiones. Basado en índice DMX-LIV.

### Endpoint
`GET /api/v1/scores/livability?lat={lat}&lon={lon}&radius_m={100-2000}`

### Request schema
```ts
const livabilityRequest = z.object({
  lat: z.number().min(-90).max(90),
  lon: z.number().min(-180).max(180),
  radius_m: z.number().int().min(100).max(2000).default(500),
  country_code: z.string().length(2).optional().default('MX'),
  persona: z.enum(['quiet','nightlife','family','fitness','remote_worker','investor']).optional(),
});
```

### Response schema
```ts
{
  lat: 19.4326, lon: -99.1332, radius_m: 500,
  score: {
    livability: 82.3,
    breakdown: {
      walkability: 85.2,  // N08
      safety: 78.1,       // F01
      ecosystem: 88.5,    // N01
      water: 72.4,        // N07
      schools: 81.0,      // H01
      health: 76.8,       // H02
      senior_friendliness: 68.2,  // N10
      transit: 92.1,      // F02
      crime_trend: 75.5,  // N04
    },
  },
  confidence: 'high',   // 'high' | 'medium' | 'low' | 'insufficient_data'
  currency_context: { currency: 'MXN' },
  disclaimer: 'Metodología pública en desarrollosmx.com/metodologia',
  version: '1.0.0',
  generated_at: '2026-04-17T10:15:00Z',
}
```

### Target clientes
- **Portales**: Inmuebles24, Vivanuncios (widget en ficha).
- **Fintechs**: Creditas, Coru, Yotepresto (scoring crediticio adicional).
- **Apps movilidad**: Uber, Cabify (heatmap destino).
- **Gobierno**: SEDUVI, INVI (planeación).
- **Aseguradoras**: GNP, MetLife (pricing pólizas).

### Pricing
| Plan | Precio/mes MXN | Queries/día | Incluye |
|---|---|---|---|
| Free | $0 | 100 | score + confidence |
| Starter | $5,000 | 10K | + breakdown + components |
| Pro | $25,000 | 100K | + historical + trends + bulk 100/call |
| Enterprise | Custom (desde $50K) | Ilimitado | SLA 99.9%, dedicated, custom endpoints |

### SLA H1
- p95 < 200ms.
- Uptime 99.5% (starter), 99.9% (pro+enterprise).

## Producto estrella MVP — DMX Estimate (I01)

### Overview
AVM (Automated Valuation Model) mexicano. Inspirado en Zillow Zestimate + HouseCanary Value by Conditions.

### Endpoint
`POST /api/v1/estimate`

### Request
```ts
{
  address?: string,
  lat: number, lon: number,
  m2_totales: number,
  m2_construccion?: number,
  recamaras: number,
  banos: number,
  estacionamientos?: number,
  amenities?: string[],
  age_years?: number,
  tipo_propiedad: 'vertical'|'horizontal',
  condition?: 'nueva'|'remodelada'|'seminueva'|'usada'|'regular',
  country_code: 'MX',
}
```

### Response
```ts
{
  estimate: 4_850_000,
  range_low: 4_620_000,
  range_high: 5_100_000,
  currency: 'MXN',
  confidence: 0.78,              // 0-1
  comparables: [
    {address, m2, precio, distance_m, weight},
    // ...
  ],
  adjustments: {
    amenity_premium: 0.08,
    age_discount: -0.04,
    location_score: 0.15,
  },
  market_context: {
    zone_momentum: 0.78,
    zone_livability: 82.3,
    absorption_6m: 0.20,
  },
  version: '1.0.0-h1-linear',   // h2: gradient-boosting, h3: deep-learning
  generated_at: '...',
}
```

### Entradas (47 variables, 12 fuentes)
- `market_prices_secondary` (comparables secundarios)
- `project_scores` (primaria)
- `zone_scores` (amenities + livability)
- `macro_series` (inflación, tasas)
- `geo_data_points` (DENUE, GTFS, Atlas)
- `operaciones` (calibración transacciones — Tier 4)
- Características propiedad
- Condiciones específicas

### 3 fases de desarrollo
- **H1**: Regresión lineal + comparables ponderados por distancia y similitud.
- **H2**: Gradient boosting (XGBoost) con 47 features.
- **H3**: Deep learning calibrado con transacciones reales (requiere ≥100 ventas).

### Pricing
| Plan | Precio MXN | Consultas |
|---|---|---|
| Free | $0 | 5/mes por usuario |
| Pro | $2,999/mes | 500/mes |
| API Starter | $15,000/mes | 10K/mes |
| API Pro | $30,000/mes | 100K/mes |
| API Enterprise | $50,000+/mes | Ilimitado + SLA + calibration custom |

### Target
- Bancos y fintechs (originación hipotecaria).
- Valuadores (pre-screening).
- Portales (widget valor estimado).
- Compradores individuales (Free tier + Pro subscription web).

### Fase
FASE 12 (modelo H1) + FASE 23 (API pública B2B).

## Resolución de la tensión "7 productos o 8"

El briefing dice **"7 productos B2B"** pero DMX Estimate también vende bajo el mismo paraguas. Resolución:

- **7 productos "licenciables por zona/empresa"** (DMX Livability, Momentum, Risk, Site Selection, Market Reports, COMPASS H3, Knowledge Graph H3) — suscripciones enterprise.
- **DMX Estimate** es producto **MVP independiente** con doble pricing (Free+Pro consumer web + API B2B).

Se mantiene "7 productos" como los licenciables corporate, y **DMX Estimate es el producto estrella MVP** paralelo (documentado aquí como `★`).

La API externa v1 expone los 7 + estimate = 8 endpoints monetizables. El lineup comercial dice "DMX: 7 productos B2B + DMX Estimate (la fundación pública)".
