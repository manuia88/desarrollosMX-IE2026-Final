Middleware base (FASE 02):
- `publicProcedure`: sin auth.
- `authenticatedProcedure`: requiere `ctx.session.user`. Rate limited 600/min.
- `asesorProcedure`: authenticatedProcedure + `ctx.profile.rol in ('asesor','master_broker','admin','superadmin')`. Rate limited 300/min.
- `developerProcedure`: rol `desarrollador` o `admin`.
- `contadorProcedure`: rol `contador` o feature `contabilidad_access` override.
- `compradorProcedure`: rol `comprador`.
- `adminProcedure`: rol `admin` o `superadmin` + logs en `admin_actions`.
- `featureProcedure(key)`: verifica `check_feature_limit(userId, key)`.

Convención: todo `input` es Zod schema nombrado `<procedure>Input`. Toda `mutation` sensible crea row en `audit_log`.

## Middleware stack detalle

```ts
// server/trpc/middleware.ts
export const rateLimited = t.middleware(async ({ ctx, path, next }) => {
  const key = ctx.session?.user?.id ?? ctx.ip;
  const ok = await checkRateLimitDb(key, path, 300);
  if (!ok) throw new TRPCError({ code: 'TOO_MANY_REQUESTS' });
  return next();
});

export const authenticated = t.middleware(({ ctx, next }) => {
  if (!ctx.session?.user) throw new TRPCError({ code: 'UNAUTHORIZED' });
  return next({ ctx: { ...ctx, userId: ctx.session.user.id }});
});

export const requireRole = (roles: string[]) => t.middleware(({ ctx, next }) => {
  if (!roles.includes(ctx.profile.rol)) throw new TRPCError({ code: 'FORBIDDEN' });
  return next();
});

export const requireFeature = (key: string) => t.middleware(async ({ ctx, next }) => {
  const limit = await checkFeatureLimit(ctx.userId, key);
  if (!limit.enabled) throw new TRPCError({ code: 'FORBIDDEN', message: `feature_gated:${key}` });
  if (limit.remaining !== null && limit.remaining <= 0) throw new TRPCError({ code: 'PAYMENT_REQUIRED', message: `limit_exceeded:${key}` });
  return next({ ctx: { ...ctx, featureLimit: limit } });
});
```

## Notas / disclaimer

- Todo procedure `mutation` sensible (operaciones, pagos, admin) inserta en `audit_log` vía el middleware `auditTrail` envolvente.
- Los inputs Zod son la fuente única de verdad: se comparten con forms react-hook-form (`zodResolver`) y con validación server-side (tRPC).
- Tipos inferidos: `type ProcedureOutputs = inferRouterOutputs<AppRouter>` exportado para uso cliente.
- Routers se componen en `server/trpc/root.ts`:
  ```ts
  export const appRouter = router({
    auth, profile, contactos, busquedas, captaciones, tareas, operaciones,
    desarrollos, marketing, estadisticas, dashboard, developer, contabilidad,
    comprador, admin, ie, ai, billing, notifs, platform,
  });
  ```
