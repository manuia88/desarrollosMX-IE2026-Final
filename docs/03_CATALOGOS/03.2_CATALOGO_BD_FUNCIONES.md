**Regla ADR-009 aplicada en toda función SECURITY DEFINER:**
```sql
create or replace function public.<fn>(...) returns <type>
language plpgsql
security definer
set search_path = public, pg_temp
as $$
begin
  if auth.uid() is null then
    raise exception 'auth_required' using errcode = '42501';
  end if;
  -- ...
end;
$$;
```

## Notas / disclaimer

1. Todas las funciones SECURITY DEFINER son `set search_path = public, pg_temp` para evitar search_path injection (hardening estándar Supabase 2026).
2. El conteo ~58 es planeado. Cada fase agrega funciones incrementalmente. La migration final del rewrite debe verificar con:
   ```sql
   select routine_schema, routine_name, security_type
   from information_schema.routines
   where routine_schema in ('public','private')
   order by routine_schema, routine_name;
   ```
3. Las funciones IE (`5.*`) son invocadas principalmente por workers cron (ver catálogo 03.7) y por triggers de cascada (ver catálogo 03.3).
4. Las funciones de contabilidad (`4.10`-`4.13`) usan extensión `pg_net` para llamar endpoints externos (Facturapi, Banxico) de forma asíncrona.
5. Cambios a funciones SECURITY DEFINER requieren aprobación explícita en code review (ADR-009).
